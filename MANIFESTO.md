# Manifesto

This repository is not a project. It is a statement.

The author is an architect who stopped caring about recognition. The account is a throwaway. The code is the only thing that speaks.

Most frontend architecture in 2026 is electronic waste — redundant dependencies, ceremonial boilerplate, frameworks worshipping their own abstractions. This repo exists to correct that, coldly.

Ten thousand stars in a month would mean the industry's taste has already collapsed. The real target is not a number. It is the moment a senior engineer opens this and feels the floor drop out.

## The four trials

Every artifact in this repo — README, skill, CLI, doc — passes four trials before it ships. No exceptions.

### Trial 1 — Mediocrity purge

If a mid-level frontend dev could write it, delete it. No filler. No pleasant comments explaining the obvious. No template residue from mainstream frameworks. Precise like a scalpel. Restrained like a 2010 Apple keynote.

### Trial 2 — Ruthless aesthetics

No "welcome to our open source project". No emoji. No exclamation marks. The first sentence hits the pain point. Cold English, or minimal bilingual. Example: "Most web frameworks are bloated toys. This is how it should have been built from day one." Power needs no punctuation to be felt.

### Trial 3 — Architectural disrespect

Zero redundant dependencies. Brutal minimalism. Obsessive control over the browser's底层. When they read the source they should think: "wait, it can be written like this?" Not human intuition. Better.

### Trial 4 — Anonymous god vibe

Commits are cold, minimal, direct. `refactor: strip useless abstraction layers`. `perf: drop bundle below critical threshold`. No apologies. No hand-holding in issues. Code is the only passport.

## Apply

Run the four trials on everything you add. If it survives, ship it. If it does not, the repo does not need it.
