一、核心认知

三座大山

痛点本质不解决会怎样
Token 诅咒所有 API Schema 塞进 System Prompt成本失控，长任务算力烧穿
上下文膨胀注意力稀释，"迷失在中间"推理暴跌，越做越错
黑盒不可控纯 API 封装报错难自查Agent 无法自我纠错

三大解法支柱

  • 渐进式披露 平时只持极简索引；任务到来按关键词按需加载 SKILL.md，用完即抛。
  • 操作诚实 CLI 是确定性接口。Agent 跑错，人类可复制命令改参复现——人/Agent 共享操作面。
  • 三层解耦 MCP(连接/认证)·CLI(原子执行)·Skills(SOP 编排)，各管一层。

工程范式命名

Terminal-Native Agent Scaffolding（终端原生代理脚手架）· Lazy Tool Discovery（延迟工具发现）· 双代理架构（规划/执行分离 + 自适应上下文压缩）。
