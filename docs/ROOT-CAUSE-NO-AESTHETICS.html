生成站点「无美学」根因诊断

Meoo 内测反馈 · 流程完美，产物像 PPT · 2026-07-21

一句话根因：我们把「美学」当成了文字风格描述传给平台，而平台生成模型无法把文字翻译成复杂前端代码。正确做法是把「美学」作为可运行的基底代码强制注入生成流程。

四个断点

断点 1

案例召回只"展示"、不"注入代码"

§1.6 仅把 INDEX.json 的文字元数据（"字符级动效·磁吸光标"）展示给用户；preview 字段的 raw HTML 链接只是展示用，技能无任何指令让 agent 去 fetch 源码作基底。模型拿到词、自己"实现"→ 复杂动效被静默丢弃。

断点 2

Step 3 / 4 是"自由设计"，无"强制套用骨架"

Step 3「定义色彩/字体/选风格」、Step 4「关键动效点」都是让 agent 从零设计，没有"以选中 site/index.html 为不可变骨架、仅替换品牌变量"的硬指令。美学完全看模型临场发挥。

断点 3

broker 守门只查"一致性"，不查"视觉交互密度"

守门 8 维的"视觉一致"仅查色板/字体/质感是否对齐，不查"有没有动画、有没有交互、视觉锤是否到位"。朴素静态站只要颜色对就放行——美学维度没人守。

断点 4

平台生成机制错配（Meoo）

Meoo 是云端 AI 开发工具，不会自动加载仓库 templates/catalog 代码。只给文字描述时它用默认朴素模板。必须把 aurora 完整 HTML 源码塞进 prompt 让它"改"而非"造"。

解决方案构想

A 治本

基底代码注入机制

§1.6 后加硬指令：选中模板必须读取 site/index.html 完整源码作不可变骨架，仅替换 {{BRAND_NAME}}、--accent 占位符，禁重写结构/动画/交互；Case Recall 改"内联基底代码块"而非只给链接。

B 分平台

分平台基底交付格式

云端 agent（Meoo/Coze）可能禁出网 / 难处理长 HTML → 在 builder.md 内联去品牌化基底骨架或提供 base.min.html 轻量版；指令升级为"复现等效视觉与交互（在你的框架内）"。

C 兜底

美学守门门禁

broker 守门 8 维外新增「视觉交互密度」校验维：检查产出含 @keyframes/IntersectionObserver/自定义 cursor；缺失核心动效打回。Step 6 加美学自检清单。

D 验证

回归测试

把本次 Meoo 失败做成回归用例：下次跑激活语+指定 aurora 基底，检查产出是否含动画/交互，不通过则方案未生效。
