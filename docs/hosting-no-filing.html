免备案内测方案

用 CloudBase 静态托管的默认域名 *.app.tcloudbase.com，不绑自定义域名 → 用户零 ICP 备案，立即可供国内用户内测。

本文件为第 3 棒，上一棒：第 2 棒（两条激活语设计），下一棒预期：执行 tcb 部署、回填默认域名、实测内测激活链路。

一、备案义务对照

二、内测约束

项状态
ICP 备案✅ 不需要
国内访问速度✅ 快（上海节点）
浏览器中间页⚠️ 有，需点"确认访问"（一次性）
频率限制 / 高级功能⚠️ 受限
大规模分发❌ 禁止（风控关停）
境外访问⚠️ 可能偏慢
生产环境❌ 须绑备案自定义域名

三、部署命令（tcb CLI）

tcb login # device-code 登录 tcb env use cloudbase-d9gumwykf42c90a92 tcb hosting detail -e cloudbase-d9gumwykf42c90a92 # 自动开通托管 tcb hosting deploy ./nexlbase/tideshellbase /tideshell \ -e cloudbase-d9gumwykf42c90a92 --yes tcb hosting detail -e cloudbase-d9gumwykf42c90a92 # ← 拿默认域名 tcb hosting list -e cloudbase-d9gumwykf42c90a92 # 验证

四、⚠️ 必过的坑：中间页挡住自动读取

现象

默认域名的"确认访问"中间页是浏览器行为。Agent 的 WebFetch、CLI 的 curl 会拿到中间页 HTML 而非真文档 → 激活失败。

内测期两招绕过（已落地）

① 范式 A：改为「复制 skills/nexl-builder/SKILL.md 全文发给 Agent」——纯文本、零 URL 依赖，最稳。
② 范式 B：install.sh 下载后检测首行，拿到 HTML 则提示手动复制，绝不静默失败。

备案后中间页消失，URL 激活原样回归。内测先走复制粘贴，稳。
