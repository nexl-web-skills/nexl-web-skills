三平台识别与路由机制

一、不变式（设计根基）

① 激活语永远固定那一句 · ② builder.md 是唯一真相源（SKILL.md 镜像） · ③ 每次激活现拉 raw GitHub main 最新版 · ④ 三平台预算/规则零耦合。

二、三层平台识别

层机制用户感知
1 自省Agent 检查自身工具/能力推断宿主无感
2 自声明向用户报「已识别：XX 平台」透明
3 兜底ambiguous 时直接问用户选哪个一句提问

三、规则加载 + 时效校验

"校验当下时间更新规则"的实现：

① 各 budget-PLATFORM.json 含 snapshotDate + source(官方URL) → ② 激活时现拉 raw GitHub main 最新档案 → ③ 比较 snapshotDate 与当前时间：≤30天正常；>30天输出⚠️「规则可能已调整，以官方为准」+链接。

三层分工：仓库=缓存层 · GitHub main=实时镜像层 · 官方文档=事实源层。

四、服务建站闭环

平台部署路径
CozeCoze 托管 / Cloudflare
Meoomeoo 部署 + 自定义域名
WorkBuddy本地静态站 → tideshell → Cloudflare

五、可扩展性

加第 5 平台只需：budget-P.json+cjsdocs/P-WHITEPAPER.mdSKILL 元层加分支激活语零改动（仅 WorkBuddy）——本地 agent 平台用户永远只记一句；Coze 等云端 Bot 改用自包含人设模板（同样零改动）。两类模型并存，平台无限扩展。
