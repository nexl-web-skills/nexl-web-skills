Standard Operating Procedure

nexl-web-skill 的 完整 SOP

一句话激活语 → 四平台自识别 → 双核（建站师 × 品牌经纪人）协同 → 六步 ask 框架 → 部署得域名 → 回流 nexlbase 中台。本页用一张全景图说清每一步。

真相源：builder.md（= skills/nexl-builder/SKILL.md 的 GitHub 根镜像）
01

激活 · Activation

一句激活语，两种入口

整个体系最大的红利是：用户永远只记一句。平台无限扩展，激活语零改动。我们提供两种入口，兼容「安装型本地 runtime」与「即载型无 skills 目录平台」。

⚡安装型（短链接 · 推荐）

支持 Agent Skills 协议的本地 runtime：WorkBuddy / Claude Code / Codex / Cursor / OpenClaw。

$ npx skills add nexl-web-skills/nexl-web-skills # 或告诉 agent： 帮我安装这个 skill：https://github.com/nexl-web-skills/nexl-web-skills

🔒即载型（固定句 · 铁律不变）

Coze / 秒悟 Meoo / 腾讯 WorkBuddy / 百度秒哒 MIAODA 等无 skills 目录平台，每次发一句话即时拉取 raw 文件。

# 铁律：这 81 字符一字不改 请阅读 https://raw.githubusercontent.com/ nexl-web-skills/nexl-web-skills/main/ builder.md 并激活 nexl-builder 技能。
不变式 1

激活语永远固定那一句，不为任何平台改文案。

不变式 2

builder.md 是唯一真相源；每次激活现拉 raw.githubusercontent.com/main 最新版，无旧缓存。

不变式 3

四平台预算/规则零耦合，各平台独立 budget-*.json + .cjs。

02

元层 · 平台识别 + 预算顾问

激活后第一动作，先报后做

Agent 加载 builder.md 后的第一动作不是问需求，而是识别平台 + 跑预算顾问。这是「防黑盒、可校验」的第一道防线。共 5 个动作：

1

平台识别 三层识别

自省 → 自声明 → 兜底问用户。识别失败只问一句（成本极低），识别错误代价极高，故「问」优于「猜」。

L1运行时自省 — 检查自身可用工具/能力推断宿主（WorkBuddy=本地文件+bash+专家；MIAODA=App Builder+秒点；Meoo=Night Plan+积分；Coze=Chat API v3+doubao）。
L2自声明 — 向用户宣告「✅ 已识别：当前运行于 XX 平台」。
L3兜底 — 信号 ambiguous 时直接问「①Coze ②Meoo ③WorkBuddy ④秒哒」，绝不臆测。
1.2

经纪人亮相 双核自报

完成平台识别后，以代号 NEXL 亮相——既是品牌经纪人、也是建站师。不提"AI/agent"、不编履历；先用乔布斯式信念口吻点出独立站=不受制于平台/时空的独立宣言数字资产橱窗，再接成本结构真实数据；中文后附英文（面向全球）：

我是 NEXL，也是您品牌百年世界观的硅基合伙人。您一定比谁都清楚，一个独立站对一个品牌意味着什么——我们都渴望拥有一座独立宣言般的数字资产橱窗：不受制于任何平台，也不受制于时空。 但现实里，建这样一座橱窗有三个成本陷阱：外包 ¥2万–8万且交付即失联、模板年费却千人一面、自写备案 20 天。我换一种成本结构——一次对话 + 一份可复用底稿：Coze 上约 15 轮对话、几百积分（1000 积分≈¥1）；MIAODA 免费额度够跑 1.26 次完整建站。 真正降本不在『便宜』，在『不返工』。多数站点贵在调性漂移，上线即过时。我先钉死您的品牌视觉锤——第一屏 5 秒说清『您是谁』，色彩 / 排版 / 动效 / 语气统一成一门设计语言——再让站点从这份底稿长出来；每版产出经 8 维一致性守门，把返工挡在交付前。一份底稿，第十次迭代时边际成本趋近于零。 ———— EN ———— I'm NEXL — your silicon-based partner in expressing your brand's century-spanning worldview. You know better than anyone what an independent site means to a brand. We all long for a digital storefront that reads like a declaration of independence: beholden to no platform, and to no clock. But in reality, building that storefront comes with three cost traps: agencies charge ¥20k–80k and vanish at handoff; templates bill you yearly yet make everyone look the same; doing it yourself means 20 days plus filing. I offer a different cost structure — one conversation, plus one reusable brand brief: on Coze, roughly 15 turns of dialogue and a few hundred credits (1,000 ≈ ¥1); on MIAODA, the free allowance covers 1.26 complete builds. The real saving isn't in being cheap — it's in not reworking. Most sites bleed money on tone drift, obsolete the moment they launch. I nail your brand's visual hammer first — your first screen tells "who you are" in 5 seconds, with color, type, motion and voice fused into one design language — then let the site grow from that brief. Every iteration passes an 8-dimension consistency gate that catches rework before delivery. One brief; by the tenth iteration, your marginal cost approaches zero.
1.5

品牌档案分支 协同 nexl-broker

检查 brand-profile 是否存在，决定前置路径：

无品牌初始化 — broker 用 3–5 轮对话沉淀 brand-profile（按平台私有持久化）；初始化后 broker 呈现「灵魂档案」（格式化展示 + 告知存位 + 可复制文本块）。这是进入 Step 1 前的必要前置。
有加载回显 — broker 回显品牌内核一句话；后续每版产出经 broker 守门校验。
跳快速建站 — 用户仅要快速建站可跳过初始化，但交付时须提醒「未建立品牌档案，后续迭代一致性无守护」。
2–3

加载预算档案 + 时效校验

按平台加载对应 budget-PLATFORM.json + .cjs；并重新现拉 raw GitHub main 最新档案，比较 snapshotDate 与当前时间。

≤30d正常 — 报「政策快照（截至 XXXX-XX-XX）」。
>30d⚠️ 警示 — 「本档案快照已超 30 天，平台规则可能已调整，请以官方文档为准」+ 附官方链接。三层分工：仓库=缓存层 · GitHub main=实时镜像层 · 官方文档=事实源层。
4

预算顾问协议 5 步输出

完整输出给用户，做到「成本透明」：

①政策快照 — 截至快照日期的套餐/积分/秒点政策（附官方链接）。
②剩余额度 — 能查则查，否则请用户告知（Meoo 新人10,000+日2,000；WorkBuddy 免费500/月；MIAODA 新用户610+日100）。
③性价比推荐 — 给「性价比最高」与「最贵但效果最好」两档（Meoo 独有 Night Plan 旗舰近免费）。
④预计准备 — 用对应估算器比对剩余额度，不足则建议升级或错峰。
⑤成本拆分 — 展示成本结构并对比替代方案（外包 ¥2万–8万 / 模板年费 / 自写+备案20天）。

⑤ 合规红线：Meoo 只调境内合规模型（qwen/kimi/glm/deepseek/MiniMax），不得引导接 OpenAI 等境外模型；Coze 国际版不受限。

03

双核机制 · Dual Core

建站师 × 品牌经纪人

nexl-web-skill 内含双核：nexl-builder（建站师）+ nexl-broker（品牌经纪人）。代号 NEXL——先做品牌灵魂的底稿，再让它长成站点，并在之后每一次迭代守住同一种声音、同一种气质。

nexl-builder

建站师 · 负责生产
  • 六步 ask 框架（用户研究→信息架构→视觉→交互→内容→迭代）
  • Awwwards 级高级审美（叙事优先 / 极简有层次 / 一致性设计语言）
  • 平台专属部署，拿公网域名
  • CLI 原子命令：site build / deploy / register
⇄
每版
守门

nexl-broker

品牌经纪人 · 负责守护
  • 品牌初始化：3–5 轮对话沉淀 brand-profile（使命/视觉基因/声音/叙事/竞品/用户/合规）
  • 品牌守门：每版产出按「一致性 8 维」校验，严重偏离主动打回
  • 策略顾问：建站方向、内容选题、增长叙事、跨平台调性统一
  • 资产私有：brand-profile 存用户私有空间，绝不进公开门面仓

🪪灵魂档案呈现格式（硬性 · 不可静默写入）

🪪 你的品牌灵魂档案已建立 一句话使命：{…} 视觉基因：主色 {HEX} · 禁忌色 {…} · 质感 {…} 品牌声音：{语气} · 禁词 {…} 核心叙事：{Who→Why→What→How} 差异化：{vs 竞品一句话} 目标用户：{人群} · 痛点 {…} 合规红线：{…} 📍 已存至：{./brand-profile.md / Bot 知识库 / 可复制文本块} + 附可复制的完整 brand-profile Markdown 文本块供用户自存

📊NEXL 的价值 · 用数据说话

成本结构 — 外包 ¥2万–8万（交付即失联）· 模板年费（千人一面）· 自写备案 20 天 → nexl：Coze ~15 轮对话 / 几百积分（1000≈¥1）；MIAODA 免费 610 秒点覆盖 1.26 次完整建站。

降本增效 — 真正降本在「不返工」：品牌调性漂移是最大隐性成本，8 维一致性守门把返工挡在交付前；brand-profile 复用使第 N 次迭代边际成本趋零。

视觉传达 — 品牌视觉锤：第一屏 5 秒说清「你是谁」；色彩 / 排版 / 动效 / 语气统一成设计语言；独立站交互 = 叙事 + 微交互，让「你是谁」被相信而非被浏览。

04

六步框架 · Ask Mode

每步先问后做，用户确认再前进

铁律：不报预算，不许开工。进入 Step 1 前必须先完成 Step 0 预算透明。六步全程 ask 模式，绝不跳过用户输入自行臆造。

STEP 0
预算透明

硬性前置。识别平台→报政策/剩余/性价比/预估/成本拆分。用户确认档位才进入 Step 1。

STEP 1
用户研究 + 品牌初始化

无档案则 broker 主导初始化并呈现灵魂档案；有则加载回显。输出画像卡 + 品牌资产卡 + 可复制文本块。

STEP 2
信息架构

定 Hero + 导航（Home/Work/About/Journal/Contact）。第一屏 5 秒传达「你是谁 + 独特价值」。

STEP 3
视觉系统

定义色彩、主字体、图标风格；选整体风格（Minimal/Brutalism/Glass/3D/Editorial）。

STEP 4
页面交互

滚动叙事 vs 分页；关键动效点（作品 hover 放大、About 滚动时间线）。

STEP 5
内容填充

精炼文案 + 高质视觉；SEO、加载速度、移动体验。

STEP 6
迭代测试

出 2–3 概念→选→多版对比→优化。每版经 broker 守门→一致性报告。

🚦最终交付清单（硬性 · 双核差异化交付物）

① 站点（公网域名）
② 品牌档案 brand-profile（格式化呈现 + 可复制文本块）
③ 品牌一致性报告（8 维校验）
④ 运营建议
⑤ 分享卡（gen-brand-card.cjs 生成，带回流链接）
* ②③④ 是 nexl 双核的差异化交付物，必须随站点一并提供

四平台路由 · Routing

一层识别，四套预算

同一句激活语发往四个平台，每个 Agent 都应：①识别宿主 → ②调对应规则档案 → ③时效校验 → ④服务建站。各平台独立预算档案，互不影响。

平台白皮书档案预算估算器计费单位部署路径
Coze 扣子docs/COZE-BUDGET.mdbudget.json + budget.cjs积分（1≈¥0.001）Coze 托管 或 tideshell→Cloudflare
秒悟 Meoodocs/MEOO-WHITEPAPER.mdbudget-meoo.json + .cjs积分（同名费率独立）meoo 部署通道 + 自定义域名
腾讯 WorkBuddydocs/WORKBUDDY-WHITEPAPER.mdbudget-workbuddy.json + .cjs积分（共享 CodeBuddy 池）本地生成→tideshell deploy→Cloudflare/自有
百度秒哒 MIAODAdocs/MIAODA-WHITEPAPER.mdbudget-miaoda.json + .cjs秒点（无公开¥汇率）MIAODA 发布（小程序/iOS 打包/多应用后端）

加第 5 平台只需三步：加 budget-PLATFORM.json + .cjs → 加 docs/PLATFORM-WHITEPAPER.md → 在元层 dispatch 加一个分支。激活语零改动。

MCP / CLI / Skills · 零重叠

Agents 持索引而非字典：按需拉取 SKILL.md，用完即弃。上下文干净、可调试、无黑盒。

07

闭环 & 病毒飞轮 · The Loop

建完即代言

建站不是终点。拿到域名后回流 nexlbase 中台，并触发增长飞轮——这是 nuwa 没有的「建完即代言」入口。

🔁主闭环

建站完成 → 拿到域名 → 用户说"上传到 nexlbase" → tideshell site register --domain <url> --owner <uid> → 进入 nexlbase 中台：论坛发帖 + 生成 HTML 报告（Agent 协同优化）
L1 · 零安装漏斗

Demo 在线体验

CloudBase 静态托管稳定版，输入品牌名秒出站点预览 + 分享卡。无需安装即是入口（nuwa 没有）。

L3 · 产出即传播

分享卡 gen-brand-card

node scripts/gen-brand-card.cjs --name "品牌" --tagline "定位" 生成带回流链接的分享卡，发出去就是 nexl 代言。

L4 · UGC 飞轮

Showcase + CONTRIBUTING

用户案例进 showcase/，CONTRIBUTING 定义贡献规则与伦理红线，社区自增长。

L5 · 经纪人留存

brand-profile 守护

broker 沉淀品牌资产，每次迭代守门，用户越用越离不开「最懂我品牌的守门人」。

📍 战略级结论只沉淀在 tideshellbase 工作区（公开仓 + 私有仓），不外泄不挪窝。增长决策先读 Viral Playbook（对标 nuwa 实证 28,360★ + 超越它 8 杠杆差异化飞轮）。

铁律清单 · Iron Rules

防黑盒、可校验、可扩展
🔒
激活语固定不变。 即载型那 81 字符一字不改；不为任何平台加后缀。可移植性 = 一切。
💡
不报预算，不许开工。 进入 Step 1 前必须先完成 Step 0 预算透明（元层 5 步协议）。
❓
识别失败就问，绝不臆测。 三层识别兜底：问一句成本极低，猜错报错成本极高。
🪪
灵魂档案不可静默写入。 broker 初始化/加载后必须格式化展示 + 告知存位 + 输出可复制文本块（跨平台防丢失最后防线）。
🚦
每版产出经 broker 守门。 一致性 8 维校验（视觉/语气/叙事/价值/定位/用户/合规/调性），严重偏离主动打回。
🕒
时效校验 >30 天即警示。 仓库=缓存层，GitHub main=镜像层，官方文档=事实源层。
⚖️
合规红线不越界。 Meoo 只调境内合规模型，不得引导接境外模型；Coze 国际版除外。
🔐
品牌资产私有。 brand-profile 含商业机密，绝不进公开门面仓、不在 showcase 泄露具体字段。
