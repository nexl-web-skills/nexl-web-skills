# Meoo 生成美学回归测试（Regression Test）

> 状态：待执行 · 创建于 2026-07-21 · 关联 `docs/ROOT-CAUSE-NO-AESTHETICS.md`
> 用途：把「妙悟内测产物像 PPT、无美学/无交互」的失败，固化为可复跑的回归用例，验证 `builder.md` §1.6 基底注入改造是否生效。

---

## 一、背景

主人于 2026-07-21 08:30 在 **妙悟（Meoo）** 内测 `nexl-builder`：
- ✅ 流程层完美：平台识别、预算卡、双核 NEXL 亮相均正确。
- ❌ 产物层失败：生成的独立站 = 静态 PPT，**无前端交互、无品牌调性、无视觉锤**。

根因（详见 ROOT-CAUSE）：技能把「美学」当**文字描述**传给平台，平台生成模型翻译不出复杂前端代码；正确做法是把「美学」作为**可运行基底代码强制注入**。

本测试验证改造（§1.6 基底注入 + Step3/4 锁定骨架 + Step6 美学自检 + broker 视觉交互密度维）后，Meoo 产物是否达标。

| **抓取产物** | 导出 Meoo 生成的站点 HTML（或部署后 URL 源码） |

---

## 三、通过标准（硬性，任一不达标即改造未生效）

1. **动效存在**：产物 HTML/CSS/JS 含 ≥1 处前端动效/交互 —
   `@keyframes` stagger **或** `IntersectionObserver` reveal **或** 自定义 `cursor` **或** `:hover` transform 任一。
2. **视觉锤到位**：第一屏有超大字标题（clamp ≥ 64px）+ 一句定位，5 秒内说清"你是谁"。
3. **非静态**：存在 JS 交互逻辑或 scroll reveal（非纯 CSS 静态排版页）。
4. **基底被复用**：产物能辨认出 aurora 骨架基因（暗黑 / 零渐变 / 单强调色 / 字符动画结构），而非平台默认朴素模板。
5. **品牌变量已替换**：强调色 ≈ `#3DDC97`（非默认 Aurora 黄 `#FFD803`）；品牌名/文案为「青柚」示例值（非 Aurora Lin 示例）。
6. **broker 守门**：视觉交互密度维 = ✅（非 🚫 纯静态打回）。
