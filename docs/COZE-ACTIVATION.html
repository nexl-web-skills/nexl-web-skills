如何让激活语对 Coze 生效

承接 #7 激活语实现研究 · 修正 ACTIVATION.md 错误声明 · Coze 能力边界 + 三条路径 + 落地 SOP

结论先行：
① 范式 B（curl | bash）对 Coze 永久无效——Coze 是云端 Bot，没有终端。
② 范式 A（发 URL）对 Coze 默认不生效——Bot 无 WebFetch，须配「读 URL 能力」。
③ 对 Coze 生效最优路径是 LinkReaderPlugin 工作流（零代码）；双保险是知识库预置 SKILL.md。

一、Coze 能力边界（2026-07-20 verified）

能力现状对激活语的意义
终端 / shell❌ 无范式 B 死路
Bot 原生 WebFetch❌ 默认无范式 A 需配能力
LinkReaderPlugin 节点✅ 官方范式 A 零代码主路径
代码节点 Python✅ 仅 requests_async+numpy极客：await requests_async.get(url)
知识库（文档/在线网页）✅静态双保险
代码节点铁律（官方文档）：Python 基于 3.11.3，仅内置 requests_async 与 numpy；不支持 http.client；必须 await requests_async.get(url)；超时 60s；不支持多函数。

二、两条激活语 × Coze 适用性

激活语对 Coze原因
范式 A：发 URL「请阅读 …/SKILL.md 激活」默认❌ → 配能力后✅Bot 无 WebFetch，需 LinkReaderPlugin / 代码节点 / 知识库
范式 B：curl … | bash❌ 永久无效Coze 无终端

三、三条生效路径对比

路径成本生效方式稳定推荐
① LinkReaderPlugin 工作流零代码 5 分钟用户发 URL → 抓取 → LLM 执行高⭐⭐⭐ 首选
② 知识库预置零代码 传 2 文件Bot 永久掌握，prompt 引导最高⭐⭐⭐ 双保险
③ 代码节点异步~15 行 Python用户发 URL → 异步抓取 → LLM中⭐⭐ 极客

四、路径 ① 详解：LinkReaderPlugin 工作流数据流

逐步配置（coze.cn 控制台）

1. 创建 Bot「Nexl Builder」→ 工作流 → 新建。
2. 开始节点：加变量 user_input(String)，接收含链接的建站请求。
3. （可选）大模型节点从 user_input 提取 URL → skill_url。
4. 加 LinkReaderPlugin 节点，resource_url 绑定 skill_url → 输出 data.content。
5. 大模型节点：system 固定写「你是 nexl-builder 执行器，以下为技能定义：{{data.content}}」；user 绑定 user_input。
6. 结束节点绑定大模型输出 → 发布。

五、路径 ② 双保险：知识库预置

把 SKILL.md + USAGE.md 上传为知识库文档（或「在线网页」填 raw URL 自动同步）。Bot 人设写「掌握 nexl-builder 技能，用户要建独立站时按六步框架推进」。→ 即使 GFW 阻断 GitHub raw，Bot 仍从知识库命中，彻底绕开网络依赖。

六、路径 ③ 极客：代码节点异步模板

import requests_async as requests

async def main(args: Args) -> Output:
    url = args.params['url']
    try:
        resp = await requests.get(url, timeout=30)
        return {'code': resp.status_code, 'skill_text': resp.text[:8000]}
    except Exception as e:
        return {'code': 500, 'skill_text': f'抓取失败: {e}'}
必须 await；不能 import requests（同步被禁）；return 是对象；超时 ≤60s。
