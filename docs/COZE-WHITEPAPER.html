Coze（扣子）能力白皮书注解

给接力者说明 Coze 能做什么，并基于原始资料逐功能标注它在 nexlbase 独立站飞轮里的落点。

本文件为第 2 棒，上一棒：第 1 棒（架构白皮书 + 文档交付公约），下一棒预期：将 nexl-builder 做成可一键装进 Coze 的 Skill 包，实测 Chat v3 流式建站闭环。

一、Coze 是什么

Coze 是字节的 Agent 协作 + AI 编程平台，分国内版（扣子）与国际版两套独立环境。nexlbase 面向全球发布独立站，优先国际版；技能 ask 模式中文体验国内版更顺，部署时务必写死 Base URL，不要混用。

维度国内版（扣子）国际版
文档首页docs.coze.cndocs.coze.com
API Base URLapi.coze.cnapi.coze.com
Open API 参考coze.cn/open/apicoze.com/open/api

二、Coze 能做什么 · 逐功能注释

每个能力标注三件事：是什么能做什么在 nexlbase 的落点。

2.1 平台层能力

功能是什么nexlbase 落点
定制专属 Agent可视化/代码创建智能体用户创建"建站 Agent"，挂 nexl-builder 技能即获建站能力
Agent 技能自包含能力包（类 Anthropic SKILL.md）核心落点：skills/nexl-builder/SKILL.md 直接作为 Coze 技能导入
项目与协作多人/多 Agent 协同用户 + Agent 协同优化 HTML 报告
AI 编程模型/开发/集成/部署/运维建站产物在 Coze 内开发、部署拿域名
Open API外部系统调 Agentnexlbase 中台 / CLI 远程触发用户 Agent

2.2 核心 API 能力

能力接口nexlbase 落点
Chat v3 流式POST /v3/chat (stream)建站主通道：ask 模式逐步问、逐步产出
Chat v1 非流式POST /v3/chat短任务：生成一份报告
查询对话GET /v3/chat/retrieveCLI 异步等建站完成
消息列表GET /v3/chat/message/list把站点文件/文案回写 nexlbase
Workflow RunPOST /v3/workflow/run编排"部署→拿域名→回写"自动化
创建/发布 BotPOST /v3/bots/*前置铁律：不发布为 API 版本就调不到

2.3 认证（PAT）

Authorization: Bearer {PAT_Token}
# PAT 在 扣子平台 → 个人设置 → API Token 生成（有读写区分）

安全边界：nexlbase 中台绝不直接存用户 PAT；CLI 走用户本机环境变量 COZE_PAT。接力者不得破坏此边界。

四、四个必记的坑

① 文档是 SPA，工具抓只拿到标题，真内容浏览器看。
② Base URL 两套不能混，写死。

④ PAT 有读写区分，建站只需 chat 权限，别给全量 token。
