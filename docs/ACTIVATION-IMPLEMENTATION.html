两条极简激活语
如何实现

两条激活语不是 magic sentence，而是两个分发通道。能否"真正激活"，完全取决于目标 Agent 平台的 skill 加载机制。

诚实审计

ACTIVATION.md 声称"范式 A 已真激活、无需粘贴降级"——该声明对 Claude 系成立，对 Coze 不成立（bot 默认无 WebFetch）。install.sh 也只装壳、未注入 Agent，且双源 fallback 未实现。

两条激活语的分发链路

分平台实现矩阵

平台范式 A（发 URL）范式 B（curl）真激活所需
Claude / Codex✅ WebFetch 直读✅ cp 到 skills范式 B 补 cp 步骤
Cursor✅⚠ 复制到 rules范式 B 补 cp
Coze 云端⚠ 需读 URL 节点❌ CLI 无意义配 HTTP 节点 / 粘贴（USAGE A）
Coze API✅ system 上下文❌范式 A 方式 C

当前缺口

  • 范式 B 未注入本地 Agent（缺 cp SKILL.md 到 skills 目录）
  • 双源 fallback 未实现（install.sh 仅 GitHub raw 单源）
  • deploy / register 后端未接通（标注 not wired yet）
  • Coze 范式 A 缺"读取 URL"能力（需代码节点模板）
  • ACTIVATION.md 声明与代码现实落差（需按平台修正）
