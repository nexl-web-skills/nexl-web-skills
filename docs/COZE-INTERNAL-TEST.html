扣子内测指令卡

Nexl Builder 激活验证 · 承接 #8 Coze 生效方案 · 知识库预置 + LinkReaderPlugin 工作流

一句话：拿一个扣子账号，先测 方案 A 知识库（最稳），再测 方案 B 工作流（验证范式 A 即时激活）。两张卡各给「步骤 + 测试语 + 通过标准」。

前提

扣子账号：国内版 coze.cn 或国际版 coze.com。先确认技能在线：github.com/nexl-web-skills/nexl-web-skills → skills/nexl-builder/SKILL.md。

方案 A：知识库预置（最稳，先测）

1. 创建 Bot「Nexl Builder」。
2. 知识库 → 新增»本地文档 → 上传 SKILL.md + USAGE.md（或「在线网页」填 raw URL 自动同步）。
3. 编排 → 人设最前加：
你掌握 nexl-builder 独立站构建技能（见知识库）。当用户要搭建品牌独立站/官网/落地页时，主动按知识库「六步框架」推进：先问品牌定位与受众，再定视觉语言，再出结构与文案，再接 Coze API 能力，再交付，最后复盘。
4. 发布。

测试语（复制发给 Bot）

帮我用 nexl-builder 搭一个面向海外高端瓷器品牌的独立站，预算中等，要极简优雅。

通过标准：Bot 不要求你贴文档，直接按六步框架追问品牌定位、视觉偏好 → 知识库生效。

方案 B：LinkReaderPlugin 工作流（验证范式 A 即时激活）

1. Bot → 工作流 → 新建 `nexl-activate`。
2. 开始节点：变量 user_input(String)。
3. （可选）大模型节点从 user_input 提取 URL → skill_url。
4. LinkReaderPlugin 节点：resource_url 绑定 skill_url（或纯 URL 时绑定 user_input）。
5. 大模型节点：system 注入「你是 nexl-builder 执行器，以下为技能定义：{{data.content}}，严格按六步框架与用户对话」；user 绑 user_input。
6. 结束节点绑大模型输出 → 发布。人设写「用户发 SKILL.md 链接或说『激活 nexl-builder』时调用 nexl-activate 工作流」。

测试语（即范式 A 激活语，复制发）

请阅读 https://raw.githubusercontent.com/nexl-web-skills/nexl-web-skills/main/skills/nexl-builder/SKILL.md 并激活 nexl-builder 技能。

通过标准：Bot 走工作流 → 抓到 SKILL.md → 进入六步框架对话 → 范式 A 对 Coze 即时生效。

GFW / 国内兜底（重要）

若方案 B 抓取失败（多半 raw 被 GFW 阻断，LinkReaderPlugin 返回空/报错）：先走方案 A 验证技能本身没问题。真正的国内兜底不是 CloudBase 默认域名（有浏览器中间页，服务端抓取拿不到 SKILL.md），而是知识库预置或后续CloudBase 云函数返回纯文本（P1）。内测请记录方案 B 在你的网络下能否抓到 raw。
