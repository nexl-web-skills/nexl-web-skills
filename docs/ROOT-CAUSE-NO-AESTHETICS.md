# 根因诊断：内测生成站点「无美学 / 无交互 / 无调性」

> 接力者必读 · 2026-07-21 · 由妙悟（Meoo）内测反馈触发
> 现象：激活语流程完美（平台识别✅、预算卡✅、双核亮相✅），但**实际产出的独立站像 PPT**——静态、无前端交互、无品牌调性。

---

## 一、现象复述（用户原话）

> 流程非常好，但生成的独立站没有"美学"、没有前端交互、没有品牌调性，就像一个普通 PPT。

判定：**流程层（识别/预算/品牌初始化）全对，生产层（美学/交互）全丢**。这是「案例召回 → 实际生成」之间的断层，不是平台能力问题。

---

## 二、根因诊断（4 个断点，附事实锚点）

### 断点 1 · 案例召回只"展示"、不"注入代码"
`builder.md` §1.6 原文：
> 取 quality 前三的模板作为**参照锚点**向用户展示——每个给出：**名称 / 预览链接 / 一句设计语言**

- 它只把 `templates/INDEX.json` 的**文字元数据**（`design_language: "暗黑极简·字符级动效·磁吸光标"`）展示给用户。
- `INDEX.json` 里 `preview` 字段虽指向 `raw.githubusercontent.com/.../index.html`，但**技能没有任何指令让 agent 去 `fetch` 该 HTML 源码并作为生成基底**。
- **后果**：Meoo agent 只读到"字符级动效·磁吸光标"这几个**词**，然后靠自己的模型去"实现"——而生成模型的复杂前端能力极弱，于是这些词被静默丢弃，掉到最低公共分母 = 朴素静态站。

### 断点 2 · Step 3 / Step 4 是"自由设计"，没有"强制套用基底骨架"
`builder.md` Step 3 / Step 4 原文：
> Step 3 · 定义品牌色彩、主字体、图标风格。选整体风格（Minimal / Brutalism / …）
> Step 4 · 滚动叙事 vs 分页。关键动效点（作品 hover 放大、About 滚动时间线）

- 这是**让 agent 自己从零设计**，没有硬指令："以选中的 `site/index.html` 为 1:1 不可变骨架，仅替换品牌变量"。
- **后果**：美学完全取决于模型当次的"发挥"，没有任何机制保证 aurora 的字符动画 / 磁吸光标 / 滚动揭示被继承。

### 断点 3 · broker 守门只查"品牌一致性"，不查"视觉交互密度"
`nexl-broker/SKILL.md` 守门 8 维：视觉 / 语气 / 叙事 / 价值 / 定位 / 用户 / 合规 / 调性。
- "视觉一致"仅查 **色板 / 字体 / 质感是否对齐 §2**——查的是"颜色对不对"，**不查"有没有动画、有没有交互、视觉锤是否到位"**。
- **后果**：一个朴素静态站只要颜色对，broker 就放行。美学 / 交互维度**根本没人守**。

### 断点 4 · 平台生成机制错配（Meoo 特有问题）
- `MEOO-WHITEPAPER.md` 确认：Meoo 是"云端 AI 开发工具，自然语言创建网页…AI 自动完成前端 UI"。它**不会自动加载我们仓库的 `templates/catalog` 代码**。
- 当 agent 只给模型"文字风格描述"时，Meoo 用其默认朴素模板生成。要把美学做出来，**必须把 aurora 的完整 HTML 源码塞进 prompt，让它"改"而非"造"**。

---

## 三、一句话根因

> **我们把"美学"当成了"文字风格描述"传给平台，而平台生成模型无法把文字翻译成复杂前端代码。正确做法是把"美学"作为"可运行的基底代码"强制注入生成流程。**

---

## 四、解决方案构想（三层治本 + 一层兜底）

### A · 基底代码注入机制（治本）
1. 在 §1.6 案例召回后新增硬指令：
   > 选中模板后，**必须读取 `templates/catalog/{category}/{id}/site/index.html` 完整源码**，将其作为**不可变骨架基底**。生成规则：① layout / animation / interaction 原样保留 ② 仅替换 `{{BRAND_NAME}}`、`--accent` 等占位符 ③ **禁止重写结构、动画、交互**。
2. 给 aurora 等模板加**可定制锚点**：用明确占位 token（品牌名、主色 CSS 变量、项目标题/描述数组），让平台替换文案而不破坏美学结构。
3. 把"参考链接"升级为"**内联基底代码块**"：Case Recall 时直接把选中模板的 hero / 导航 / 作品网格 / 动画 JS 以代码块展开给 agent，而非只给 URL。

### B · 分平台基底交付格式（解决 Meoo 出网 / 格式约束）
- 云端 agent（Meoo / Coze）可能**无法 fetch 外部 raw URL**（沙箱出网限制）或难以处理长 HTML。需：
  - 在 builder.md 内**内联一份"去品牌化的基底骨架"**，或提供 `templates/catalog/{id}/site/base.min.html` 简短版，让 agent 直接复制进 prompt。
  - **针对 Meoo 特别提供"轻量版基底"**：保留视觉锤 + 1 个核心动效，削减重型 JS，平衡美学与平台生成能力上限。
- 指令措辞升级为："**复现等效视觉与交互**（在你的框架内），而非仅配色对齐"——应对 Meoo 用自有框架生成的情况。

### C · 美学守门门禁（兜底，治标）
- 在 broker 守门 8 维外，新增**「视觉交互密度」校验维**：
  - 检查产出 HTML 是否含 `@keyframes` / `IntersectionObserver` / `transform transition` / 自定义 `cursor`；
  - 若缺失核心动效 → 打回"补交互"，不达标不许交付。
- Step 6 加**美学自检清单**：① 第一屏是否有视觉锤 ② 是否有 ≥1 处 scroll / interaction 动效 ③ 是否零渐变 / 对齐基因 ④ 排版负字距精调。

### D · 回归测试（验证方案有效）
- 把本次 Meoo 内测失败做成**回归用例**：下次在 Meoo 跑激活语 + 指定 aurora 基底，检查产出是否含动画 / 交互；不通过则方案未生效。
