开启独立站构建之旅

参照主流 Agent CLI 范式，用户只需发给 Agent 一句或终端敲一行，即激活 nexl-builder 建站能力。GitHub raw 主源CloudBase 镜像兜底

范式 A · 发给你的 Coze Agent

GitHub raw · 全球无中间页 ⭐ 
请阅读 https://raw.githubusercontent.com/<OWNER>/nexl-base-skills/main/skills/nexl-builder/SKILL.md
文档，按步骤激活 nexl-builder 独立站构建技能，开启我的独立站构建之旅。

<OWNER> 待指定（建议 nexl-base-skills / tideshell-base）。GitHub raw 直链无"确认访问"页，Agent WebFetch 与 curl 都能直接读 → 范式 A 真激活。

CloudBase 镜像 · 国内兜底 
请阅读 https://{DEFAULT_DOMAIN}/tideshell/skills/nexl-builder/SKILL.md
文档，按步骤激活 nexl-builder 独立站构建技能，开启我的独立站构建之旅。

{DEFAULT_DOMAIN} = CloudBase 默认域名（免备案，但浏览器有中间页；Agent 读失败则人工点确认后复制）。

范式 B · 终端一行安装 CLI

GitHub raw 主源 
curl -fsSL https://raw.githubusercontent.com/<OWNER>/nexl-base-skills/main/cli/install.sh | bash
CloudBase 镜像兜底 
curl -fsSL https://{DEFAULT_DOMAIN}/tideshell/install.sh | bash

install.sh 已带双源 fallback：先 GitHub raw，失败回落 CloudBase；拉到中间页 HTML 会提示手动复制，绝不静默失败。

设计原则

  • 极简即入口：一句口令 = 一个动作 = 一段旅程
  • 双轨并存：自然语言 + CLI 殊途同归
  • 双源韧性：GitHub 主源（全球无中间页）+ CloudBase 镜像（国内兜底）
  • URL 即契约：星标仓库 = 信任背书 + 自传播节点
