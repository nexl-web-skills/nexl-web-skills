二、同类项目架构事实

① Hugo Themes（gohugoio/hugoThemesSiteBuilder）

贡献架构 = 索引式：Fork → 在 themes.txt 按字典序加【一行外部仓库 URL】→ PR → Netlify preview 校验 元数据 = 强制 theme.toml（"Only theme.toml accepted, not yaml/json"） 截图 = 强制 /images/screenshot.png(≥1500×1000) + tn.png(≥900×600)，3:2 重建 = 每天 UTC 00:00 定时自动重建站点 LICENSE = 强制开源，侵权 "rejected without further discussion"

② learning-zone/website-templates（DeepWiki）

贡献架构 = monorepo 式：Fork → 改 → PR，代码直接进主仓 标准结构 = index.html + README.md + CSS + License；命名 descriptive-hyphenated 索引方式 = 提交时手动更新主 README 索引表（[序号]|名称|描述|demo） Review = 代码质量/原创性/响应式/浏览器兼容/文档/许可

③ 我们的定位

= monorepo 式（同 ②），但更结构化： • template.json（机器可读，agent 友好）+ 总 INDEX.json（全局索引，Hugo/② 无对等物） • CI 直校验字段（优于 Hugo 仅靠 Netlify preview） • 中英双语 README（符合 Hugo「至少英文翻译」要求） 结论：monorepo 式对集体资产聚焦合理，无需改 Hugo 索引式；但漏了 Hugo 的截图规范 + 自动重建。

三、差距清单（逐项可复核）

G1
🔴 无自动画廊页（gallery）
全球用户看不见「集体资产橱窗」，与定位矛盾；只有 raw.githubusercontent 链接。
事实：GitHub Pages 官方支持从 /docs 或 Actions 发布；Hugo 有 themes.gohugo.io 橱窗。
G2
🔴 INDEX.json 靠「维护者手动重建」
CONTRIBUTING 写「维护者合入后手动重建」，单点瓶颈；维护者疏忽即停更。
事实：Hugo 用每天 UTC 00:00 定时自动重建站点。
G3
无 PR 模板
贡献者开 PR 无勾选引导，易漏 LICENSE / 自测步骤。
事实：GitHub 官方 .github/pull_request_template.md 标配；Hugo/website-templates/Ant Design 全用。
G4
无 Issue 模板
全球用户报 bug / 申请新类目无结构化入口。
事实：官方 .github/ISSUE_TEMPLATE/；同类项目均分 bug/feature/模板申请。
G5
无 CODE_OF_CONDUCT
多元贡献者缺行为公约保障。
事实：官方社区健康文件核心项（Contributor Covenant 2.1）。
G6
无 CODEOWNERS
任何人可提 PR 改 skills/、builder.md、INDEX.json，CONTRIBUTING 的「不收 SKILL.md PR」无机制强制。
事实：官方推荐 CODEOWNERS 保护关键路径（workflows/Dockerfiles/infra）。
G7
无截图规范
画廊页若有也会空 / 不一致。
事实：Hugo 强制 screenshot(1500×1000)+tn(900×600) 3:2。

四、我们做对的地方（事实印证，保留）

✅ monorepo 式（同 website-templates，适合集体资产） ✅ template.json + 总 INDEX.json（比 Hugo theme.toml 更机器可读） ✅ community-check.yml 字段校验（优于 Hugo 仅靠 preview） ✅ 中英双语模板 README（符合 Hugo 英文翻译要求） ✅ MIT LICENSE 文件已存在 ✅ author=GitHub handle 署名（荣誉归贡献者）

五、可执行改进路线图（待拍板，未执行）

P0 — 补齐官方健康社区标配

1. 加 .github/PULL_REQUEST_TEMPLATE.md（拆「模板贡献」「案例贡献」两模板 + 勾选） 2. 加 .github/ISSUE_TEMPLATE/（bug_report / template_request / doc_improvement） 3. 加 CODE_OF_CONDUCT.md（Contributor Covenant 2.1 官方原文） 4. 加 .github/CODEOWNERS：skills/、builder.md、templates/INDEX.json 需核心团队审核

P1 — 解决体验与自动化致命缺口

5. 加 scripts/build-gallery.cjs：读 INDEX.json → 生成 docs/gallery/index.html；配 GitHub Pages 从 /docs 发布 6. 改 community-check.yml 或新 rebuild-index.yml：PR 合入 main 时自动重建 INDEX.json（消 G2 单点） 7. template.json 加 preview_image 字段 + 截图规范（1500×1000 / 900×600，3:2，对齐 G7） 架构选型：保持 monorepo 式（事实支持适合集体资产），不改 Hugo 索引式； 重点补「看得见（gallery）+ 自动跑（CI 重建 INDEX）+ 守得住（CODEOWNERS/PR 模板）」。
审计一句话结论：通道已通，体验与治理还停留在「内部接力」——缺画廊页让全球看不见资产、缺 CODEOWNERS/PR 模板让治理靠自觉。补齐 P0+P1 七项，即达 Hugo Themes 级健康共建标准。
