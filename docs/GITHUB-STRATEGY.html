一、技术事实：GitHub raw 直链 vs CloudBase 中间页

维度GitHub rawCloudBase 默认域名
中间页❌ 无（直链真文件）⚠️ 确认访问页挡 Agent
Agent WebFetch✅ 直接读❌ 拿到中间页 HTML
终端 curl✅ 直接拉❌ 需人工点确认
全球可达✅ 全球 CDN⚠️ 仅上海节点
国内 GFW⚠️ 偶发被墙✅ 国内直连稳
信任背书✅ Star = 社交证明❌ 无

结论：GitHub 在"可用性 + 增长"双杀；唯一短板是国内 GFW → 解法 = 双源 GitHub 主源 + CloudBase 镜像兜底。

双源架构：Agent 优先读 GitHub raw，失败自动回落 CloudBase 镜像
