关键教训（给下一位接力者）

  • 建组织必须有 admin:org，否则只能网页 —— 别浪费时间调 API
  • transfer 遇 503 是抖动不是拒绝 → 直接重试，勿疑权限
  • git push 遇 SSL_ERROR_SYSCALL → 先 commit（本地已安全），再 sleep 重试
  • 接手旧目录先 git remote -v + git log 核对，别盲推
  • 门面分离要贯彻：公开仓只露冷峻门面，内部文档留私有仓
