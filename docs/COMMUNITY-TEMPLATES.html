〇 · 为什么是现在 / Why now

用户被激活后，最自然的一句话是：「给我看看你们做得最好的站长什么样。」

nexl 今天只有 showcase/——那是成品案例（别人建完的站，活广告）。但缺 templates/——那是可复用的起点资产（代码/配置，拿来即建）。

showcase/解决「信不信」——social proof 活广告
templates/解决「快不快」——asset reuse 集体资产

一 · GitHub 创始人的眼睛 / Through the founder's eyes

Tom Preston-Werner 把 GitHub 做成「社交编程（Social Coding）」而非代码托管。三条刻进骨子里的原则，是本体系的设计地基：

PR 是协作单元不要求先成朋友，只要求合规 PR。低摩擦 > 高门槛。
文档是项目的一半README / CONTRIBUTING 是门面，三分钟上手决定生死。
贡献者=维护者署名即归属，star 归贡献者，资产属于集体。
资产化思维：模板不是代码，是集体独立站数字资产。一个人写的极简作品集，下一千人拿去长出自己的品牌。

二 · 目录结构 / Repository layout

nexl-web-skills/ ├── showcase/ # 成品案例（活广告，已存在） ├── templates/ # ★ 新增：可复用模板库（集体资产） │ ├── INDEX.json # 机器可读总索引（agent 激活后加载） │ ├── _template/ # 贡献者复制此骨架 │ │ ├── template.json # 元数据 schema（必填） │ │ ├── README.md # 人类说明（中英） │ │ └── site/ # 可运行站点（index.html 起） │ └── catalog/ # 按用途分类（人类浏览） │ ├── personal-ip/ brand-site/ portfolio/ │ └── landing/ shop/ ├── skills/ docs/ CONTRIBUTING.md

分类用 metadata 不靠软链：物理目录按 category 分，平台适配写在 template.json 的 platforms，INDEX 据此多维检索。

三 · 模板元数据 schema / The template.json contract

每个模板必须有一份 template.json——这是 agent 调用时的唯一真相源：

{
  "id": "aurora-minimal",
  "name": "Aurora · 极简作品集",
  "author": "github-handle",
  "category": "portfolio",
  "platforms": ["coze","meoo","workbuddy","miaoda"],
  "tags": ["minimal","dark","typography"],
  "ratings": { "quality": 5, "cost_efficiency": 4, "stability": 5 },
  "cost_estimate": { "coze_credits":300, "meoo_credits":80,
                     "workbuddy_credits":50, "miaoda_seconds":420 },
  "design_language": "暗黑极简 · 强排版 · 无渐变",
  "first_screen_pitch": "5 秒内说清你是谁",
  "files": ["site/index.html"], "preview":"https://...",
  "license":"MIT", "createdAt":"2026-07-21", "updatedAt":"2026-07-21"
}

ratings 三维对应「最优秀 / 性价比最高 / 代码最稳定」；cost_estimate 让 agent 直接报预算；platforms 让四平台只调自己能跑的。

四 · 三维评级标准 / The three-axis rating

维度含义评审看什么高分信号
quality 优秀视觉/工艺第一屏 5 秒说清「你是谁」+独特价值；设计语言统一；Awwwards 级视觉锤清晰、零廉价感
cost_efficiency 性价比对应平台成本cost_estimate vs 同类；低消耗达高质感更少积分/秒点产更高质感
stability 稳定代码质量零/最小依赖；响应式；跨平台一致；无 console 报错即拷即用、无破绽
agent 调用规则：问「最优秀」→ quality 降序；「性价比最高」→ cost_efficiency 降序且 cost_estimate 最低；「代码最稳定」→ stability 降序。三问三答，无需人工挑。

五 · 案例调用索引 / Case-recall for agents

被激活的 agent 如何在四平台内调出最优模板——这是 NEXL 用户远超普通入口人群的关键杠杆。

5.1 SOP 新增「①.6 案例锚定」

平台识别（①）之后、预算顾问（③）之前：加载 templates/INDEX.json，按当前平台 + 用户行业/风格过滤，取 quality 前三作参照锚点展示；用户可指定「以某模板为基底」，从底稿 + 模板双源生长。

5.2 四平台调用方式

平台agent 如何加载 INDEX 并推荐
Coze知识库预置 INDEX.json 文本；Bot 对话第二步 read 并推荐 quality 前三
秒悟 Meoo激活语后本地读 INDEX（或 MCP）；Night Plan 零成本检索
WorkBuddy技能激活即加载 INDEX 到上下文；免费额度内无限检索
MIAODAApp Builder Skill 内嵌 INDEX；全栈原生 clone scaffold 起步

5.3 机器可读索引

templates/INDEX.json 汇总所有模板的 id/name/category/platforms/ratings/cost_estimate/preview，agent 一次加载、多维过滤；CI 在模板 PR 合入时自动重建。

六 · 共建规则 / Contribution rules

贡献路径从两套扩为三套（CONTRIBUTING.md）：

① showcase成品案例，star 归你（已存在）
② templates ★模板资产，署名归你，上 INDEX（新增）
③ 核心 skillissue 讨论，维护者实现，守单一真相源

模板贡献流程

复制 _template/ → 填 template.json → 放可运行 site/ → 写 README.md → 提 PR → CI 校验 → 维护者+社区三维评级 → 合入重建 INDEX。

伦理红线

不蒸馏真人冒充其观点（我们是 brand-profile，非名人蒸馏）；模板不含侵权素材；brand-profile 商业机密绝不进公开仓。

八 · 世界级资产标准 / What "world-class" means

灵魂可携带配 brand-profile，换品牌不换骨架
视觉锤 < 5秒第一屏说清你是谁
一门设计语言色彩/排版/动效/语气统一
零调性漂移8 维守门，合入即达标
主权独立零平台锁定，可导出自托管
即拷即用stability=5，无破绽
复利被千人复用，边际成本趋零
工艺Awwwards 级精度
当 templates/ 有一百个这样的模板，nexl 就不只是一个建站 skill——它是全球开发者的独立站数字资产交易所。
