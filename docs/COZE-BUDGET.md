# Coze 建站预算透明模块（nexl-builder × Coze 积分）

> 本文件为第 12 棒。上一棒确立「唯一激活语 + GitHub 仓库 URL 缩短 + 绝不弹拦截提示」。本棒解决一个用户真实痛点：**建站前不知道要花多少**。大多数建站工具把成本藏进黑盒，nexl-builder 反其道——**每一步、建站前就把预估积分报给用户**。下一棒预期：把本模块接入 Coze 工作流（建站首轮先输出预算卡），或做成 nexlbase 中台的可视化预算页。

## 0. 为什么这是「amazing」的差异点

用户上一棒反馈：Coze agent 反应「不够 amazing」。预算透明本身就是降维感来源——

- 模板建站工具：告诉你「年费 ¥299 起」，但不知道一次对话烧多少算力。
- 外包工作室：报价 ¥2万–8万，过程不透明。
- **nexl-builder**：建站前先说「这次对话预计消耗 ~400 Coze 积分（≈¥0.4），含 4 张配图则 ~1280 积分（≈¥1.3）」。精确到积分，用户秒懂「这比一杯水还便宜」。

这种**精确的成本确定性**，是「神级马甲」世界观里「对用户的每一分算力负责」的具体落地。

## 一、Coze 计费规则（2026-07-20 经官方 `internal_integrations_fee` 核验）

### 1.1 单位
- 计费单位 = **积分**（Coze 平台币）。
- **1 积分 ≈ ¥0.001**（由 doubao-seed-2.0-pro 输入 3.2 积分 = ¥0.0032 反推）；**1000 积分 ≈ ¥1**。
- 资源包抵扣价更低（方舟 doubao-pro-32k 输入 0.8 / 输出 2），此处取**平台直购价**作保守估算。

### 1.2 阶梯计费（关键）
按**上下文长度**分档，上下文越长单价越高（输出在跨 32K 后**翻倍**）：

| 上下文档 | doubao-pro 输入 | 输出 | 缓存命中 | DeepSeek-V3.2 输入 | 输出 |
|----------|----------------|------|----------|-------------------|------|
| [0, 32K] | 3.2 | 16 | 0.64 | 2.0 | 3.0 |
| (32, 128K] | 4.8 | 24 | 0.96 | 4.0 | 6.0 |
| (128, 256K] | 9.6 | 48 | 1.92 | 8.0 | 12.0 |

> 单位：积分 / 千 tokens。**缓存命中** = 静态前缀（如 SKILL.md、系统提示）被 Coze 缓存后按此低价计——六步建站里系统提示每轮都命中缓存，这是省钱大头。

### 1.3 生图是大变量
| 模型 | 单价 |
|------|------|
| Doubao-Seedream-5.0 | **220 积分 / 张** |
| Doubao-Seedream-4.5 | 250 积分 / 张 |

> 一次建站若生成 4 张概念图 = **+880 积分**，直接超过纯对话成本 2 倍。预算卡必须单列生图项。

### 1.4 真实消耗结构（CSDN 实测，非官方）
大模型调用 **38%** / 插件调用 **27%** / 后台任务·心跳 **22%** / 中断·调试浪费 **13%**。
→ 多数积分不是花在「对话」上，而是插件与后台任务。nexl-builder 走**知识库预置**（方案 A）而非每次运行时抓 URL，可砍掉大量后台/插件浪费。

## 二、六步建站 Token 估算模型

参数化、确定性。核心假设（写入 `budget.json`）：
- `sys_tokens = 3000`（系统提示 + SKILL.md 注入，首轮全量、之后命中缓存）
- 每轮新增输入 `fresh = 150 + (turn-1)×450`（用户新消息 + 历史增量）
- 每轮输出 `out = 600`
- 典型 6 步建站 ≈ **15 轮**对话（每步 2–3 轮 ask）

公式（每轮）：
```
turn_cost = (首轮? sys_tokens×in_rate : sys_tokens×cache_rate)   // 静态前缀
          + fresh × in_rate                                        // 新增输入
          + out × out_rate                                         // 输出
总积分 = Σ turn_cost + images × 220
```
上下文 `ctx` 随轮增长（`ctx += fresh + out`），跨档后 `in_rate / out_rate` 自动升档。

> 完整实现见 `skills/nexl-builder/budget.cjs`（`node budget.cjs` 直接跑，亦可 `--json` 给 Agent 程序化调用）。

## 五、预算透明设计原则（落地到 SKILL.md）

在 `SKILL.md` 六步框架前新增 **Step 0 · 预算透明**：

1. 用户说「建站」→ Agent 先用 `budget.cjs` 等效逻辑（或心算）估算本次对话的积分区间。
2. 输出一张**预算卡**：模型档 / 预估轮数 / 生图张数 / 预估积分 / ≈人民币 / 对比替代方案。
3. 用户确认（或选「省钱档 / 质量档 / 含生图」）→ 才进入 Step 1 用户研究。
4. 第六步迭代若超出预估轮数，主动提示「已用 X 积分，预计还需 Y，是否继续」。

> 铁律：**不报预算不许开工**。这是 nexl-builder 与黑盒工具的分界线。
