一、nuwa-skill 范式解码

1.1 为什么人家的链接那么短？

维度nuwa-skill我们（当前）
激活语github.com/alchaincyf/nuwa-skill（39字符）raw.githubusercontent.com/nexl-web-skills/nexl-web-skills/main/builder.md（81字符）
范式安装型：clone 整个仓库到 runtime 的 skills/ 目录，agent 自动加载根 SKILL.md即载型：Coze/Meoo 无 skills 目录，每次发句即时拉 raw 文件

结论：短的本质是范式差异——nuwa 走标准 Agent Skills 协议 + npx skills add 安装器，激活语是"安装指令"而非"即载文件"。我们为兼容 Coze/Meoo 必须给深层 raw 路径，自然更长。

1.2 架构（纯 GitHub 仓库，零服务端）

SKILL.md 核心资产（自包含，不接受外部 PR） · references/ 渐进披露 · examples/ 已生成产物 · LICENSE MIT · 多语言 README。

文档即技能：SKILL.md = markdown + YAML，零依赖，agent 读即得。靠 npx skills add 跨 55+ runtime 自动分发。

二、我们对齐范式的改造（不破坏现有铁律）

2.1 双激活语策略（关键，解决短链接诉求）

类型适用平台激活语
安装型短WorkBuddy / Claude Code / Codex / Cursor / OpenClawnpx skills add nexl-web-skills/nexl-web-skills
即载型铁律Coze / 秒悟 Meoo / 无 skills 目录平台请阅读 .../main/builder.md 并激活 nexl-builder 技能。

支持标准的平台用短链接，即载平台用固定兜底句——两全其美，且不违背"激活语固定"铁律（即载句一字不改；短句是新增安装型入口）。

2.2 根入口标准化

公开仓根目前是 builder.md（SKILL.md 镜像）。Agent Skills 标准期望根 SKILL.md。建议统一为 SKILL.md（即载句同步改为 .../main/SKILL.md，长度不变）。此改动影响固定激活语，需主人确认后再做（见第五节）。

2.3 模块化（已部分具备，补全即可）

SKILL.md 调度器（四平台识别 + 双核路由） → skills/nexl-builder/（建站，已有） + skills/nexl-broker/（品牌经纪人，新增） → references/（预算档案 + 品牌框架） → docs/（白皮书）。

三、双核机制：nexl-builder + nexl-broker

3.1 概念区分（原创，非蒸馏名人）

nuwa-skill我们的 nexl-web-skill
蒸馏名人思维框架（认知操作系统）沉淀品牌资产（brand-profile，品牌认知操作系统）
人物 Skill 做"视角分析"品牌经纪人做"一致性校验 + 增长叙事顾问"
对象：公众人物对象：用户的品牌（含未成名 IP）

本质不同：nuwa 让人替你思考，我们让 agent 成为唯一最懂你品牌的守门人 + 操盘手。概念、对象、产出均不同，无抄袭风险。

3.2 nexl-broker 四大能力

  • 品牌初始化：首激活用 3-5 轮对话沉淀 brand-profile.md（使命/价值观/视觉基因/品牌声音/核心叙事/竞品/用户画像/合规红线）。
  • 品牌守门：每次 builder 产出，broker 先读档案做一致性校验，偏离内核自动标注或打回。
  • 策略顾问：基于品牌定位给建站方向、内容选题、增长叙事、跨平台调性统一建议。
  • 资产私有：brand-profile.md 存用户私有空间，绝不进公开门面仓。

3.3 激活后调度流

加载 → ① 平台识别 → ② brand-profile 存在？（否→broker 初始化 / 是→加载） → ③ 预算顾问 → ④ builder 建站（每版经 broker 守门） → ⑤ 交付站点+品牌档案+运营建议。

四、侵权风险规避清单（硬约束）

  • 不复制 nuwa 任何文件原文、示例对话、评分数字、模板措辞。
  • 概念原创：我们是"品牌资产化 + 守门"，nuwa 是"名人蒸馏"——对象、方法论、产出均不同。
  • 架构范式可借鉴：自包含、渐进披露、质量守门、模板驱动 = Agent Skills 协议通用范式，非 nuwa 独创。
  • 治理借鉴通用做法：核心 SKILL.md 不接受外部 PR、附 LICENSE = 行业惯例。
  • 守门清单自研：基于品牌管理常识，不引用 nuwa 的 fidelity 维度文字。
  • 仓库自写 LICENSE（建议 MIT），不宣称源自 nuwa；如需致谢仅注"范式受 Agent Skills 生态启发"。
